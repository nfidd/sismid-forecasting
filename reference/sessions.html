Session timetable

This is the Forecasting & evaluation course, which runs in the second half of the SISMID week (Wednesday midday to Friday). The companion Nowcasting & \(R_t\) estimation course covers the first half of the week.

ImportantProposed schedule — for Sam/Nick to confirm

The day/time blocks below are a concrete proposal built from the material that currently remains in this course, using the standard SISMID session grid (Wednesday half-day from 13:30; Thursday and Friday full days from 09:00, with a mid-morning break, lunch 12:00–13:00, and a mid-afternoon break). Durations and ordering reflect the relative depth of each session’s notebook and are offered as a starting point, not confirmed timings. See Schedule notes & gaps for where the content is thin or where assets are missing.

Day 1: Forecasting concepts and models

Wednesday — half day, 13:30–17:30 (handing over from the nowcasting half of the week)

Session 0: Introduction and course overview

Wednesday: 13:30–14:00 (30 mins)

  • Introduction: the course and the instructors (10 mins)
  • Recap and handover from the nowcasting half of the week; where forecasting fits (5 mins)
  • Getting started: getting set up for this course (5 mins)
  • Introduction to the course and motivation: from surveillance data to decisions, and why we forecast (10 mins)

Session 1: Forecasting concepts

Wednesday: 14:00–15:00 (60 mins)

  • Introduction: forecasting as an epidemiological problem; what makes a good probabilistic forecast (15 mins)
  • Practice: basic forecasting with ARIMA models (40 mins)
  • Wrap up (5 mins)

Wednesday: 15:00–15:30 — break

Session 2: Improving forecasting models

Wednesday: 15:30–17:30 (120 mins)

  • Introduction: ARIMA concepts, ACF/PACF, transformations and seasonality (15 mins)
  • Practice: ACF/PACF plots, differencing, transformations and seasonal terms (95 mins)
  • Wrap up and look-ahead to evaluation (10 mins)

Day 2: Forecast evaluation and ensembles

Thursday — full day, 09:00–17:00

Thursday: 09:00–09:15

  • Day 1 review (15 mins)

Session 3: Forecast evaluation

Thursday: 09:15–12:00 (with the 10:30–11:00 break)

  • Introduction: proper scoring rules, calibration, sharpness, MAE/RMSE/CRPS (20 mins)
  • Practice: scoring forecasts, time-series cross-validation, comparing models (130 mins, spanning the break)
  • Wrap up (10 mins)

Thursday: 12:00–13:00 — lunch

Session 4: Forecasting with ensembles

Thursday: 13:00–15:00 (120 mins)

  • Introduction: why ensembles, forecast representations (samples/quantiles), linear opinion pools and Vincent averaging (20 mins)
  • Practice: building and evaluating ensembles, hubverse formats (95 mins)
  • Wrap up (5 mins)

Thursday: 15:00–15:30 — break

Session 4 (cont.): Ensembles, continued / catch-up

Thursday: 15:30–17:00 (90 mins)

  • Continue the ensembles practical and weighted/trimmed ensemble extensions
  • Catch-up buffer for evaluation and ensembles
  • Wrap up (10 mins)

Day 3: Collaborative forecasting and course summary

Friday — full day, 09:00–17:00

Friday: 09:00–09:15

  • Day 2 review (15 mins)

Session 5: Evaluating real-world outbreak forecasts

Friday: 09:15–12:00 (with the 10:30–11:00 break)

Schedule notes & gaps

The following notes flag where the proposed schedule depends on instructor judgement, and where the remaining content or assets need attention before the course can be taught end to end. For Sam/Nick to review.

Timing / pedagogy proposals to confirm

  • Whole grid is a proposal. Start/end times follow the standard SISMID grid (Wed 13:30–17:30; Thu/Fri 09:00–17:00) but the per-session splits are inferred from notebook length, not from how they have actually run. Confirm against the real room schedule.
  • Wednesday handover. Session 0 assumes a short recap handing over from the nowcasting half; agree with the nowcasting instructors how much (if any) recap is needed so participants who only attend the forecasting half are not lost.
  • Day 1 is light. Only two practicals (forecasting-concepts, forecasting-models) fit in the Wednesday half-day. forecasting-concepts is the shortest notebook (~280 lines) — there is room to expand it or to start forecast-evaluation early.
  • Day 2 buffer. A continuation/catch-up block is proposed for Thursday afternoon because forecast-evaluation (~630 lines) and forecast-ensembles (~715 lines) are the two densest notebooks and routinely overrun. If they run to time, use the block for the weighted-ensemble extensions.

Content gaps (thin for the time available)

  • forecasting-concepts is noticeably shorter than the other practicals. If Wednesday afternoon feels under-filled, candidates for expansion: more on interpreting fable output, a short exercise contrasting point vs probabilistic forecasts, or an explicit bridge from the nowcasting half.
  • hub-playground has comparatively few exercise/callout markers for its length; it may want clearer step-by-step exercises. (It depends on the toy hub sandbox — see “Other notes” — which is provided automatically in CI, so this is not a blocker.)
  • Course recap deck (closing.qmd) is the shared deck and still recaps the nowcasting half (delay distributions, \(R_t\), nowcasting). For a forecasting-only cohort those slides may need trimming or reframing.

Minor asset gaps

  • Missing sessions/slides/figures/wili_timeseries.png. Referenced (currently commented out) in improving-forecasting-models.qmd. The intended title-page time-series figure is absent from the repository. Either add the figure or remove/replace the reference before relying on that slide. This is minor — the line is already commented out, so the slide still renders.

Other notes

  • Toy hub sandbox (sismid-ili-forecasting-sandbox/). hub-playground.qmd calls here::here("sismid-ili-forecasting-sandbox"). This directory is intentionally not committed (it is in .gitignore): the deploy workflow (.github/workflows/deploy.yaml) checks out reichlab/sismid-ili-forecasting-sandbox into the repo root at build time, so the session renders fine in CI. This is not a blocker. To run the hub playground locally, clone the sandbox into the repo root first, e.g. git clone https://github.com/reichlab/sismid-ili-forecasting-sandbox from the repository root (the same path the workflow uses).
  • Unscheduled deck. introduction-to-the-spectrum-of-forecasting-models.qmd exists but is not linked from any session. It is offered above as optional Friday material; decide whether to schedule it, fold it into Session 1, or drop it.
